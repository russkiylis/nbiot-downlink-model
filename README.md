*readmeslop mk2*

Доброго времени бытия! Данный репозиторий — это реализация генератора нисходящего канала стандарта NB-IoT в MATLAB, опирающаяся на спецификации [3GPP TS 36.211](https://portal.3gpp.org/desktopmodules/Specifications/SpecificationDetails.aspx?specificationId=2425) (физические каналы и модуляция) и [3GPP TS 36.212](https://portal.3gpp.org/desktopmodules/Specifications/SpecificationDetails.aspx?specificationId=2426) (мультиплексирование и канальное кодирование).

## Краткая теория

Стандарт NB-IoT использует полосу 180 кГц, разделённую на 12 поднесущих с шагом 15 кГц. По оси времени сигнал организован следующим образом: 7 OFDM-символов составляют слот, два слота — сабфрейм, десять сабфреймов — фрейм. Ресурсная сетка представляет собой двумерное отображение сигнала в координатах «поднесущая × OFDM-символ», на котором размещаются физические каналы и сигналы.

В нисходящем канале NB-IoT определены следующие сигналы и каналы: NPSS (первичный сигнал синхронизации, формируется на основе последовательности Задова—Чу), NSSS (вторичный сигнал синхронизации, передаётся только в чётных фреймах), NPBCH (широковещательный канал для передачи системной информации MIB-NB), NPDSCH (канал передачи пользовательских данных и SIB), NPDCCH (канал управления для передачи DCI) и NRS (опорный сигнал, используемый для оценки канала). Каждому каналу в структуре фрейма отведены определённые сабфреймы: сабфрейм 0 всегда занят NPBCH, сабфрейм 5 — NPSS, сабфрейм 9 в чётных фреймах — NSSS; остальные сабфреймы распределяются между NPDSCH и NPDCCH.

## Быстрый старт

Откройте `Main.m` — в нём задаётся конфигурация и запускается генерация ресурсной сетки.

Минимальный пример:

```matlab
grid = NBIoTResourceGrid;
grid.Config.totalFrames = 20;
grid.Config.NCellID = 0;
grid.Config.startFrame = 0;

% Пользовательские данные для NPDSCH
grid.Config.Bits.NPDSCH_Codeword{3}.bits = [ones(1,100) zeros(1,100)];
grid.Config.Bits.NPDSCH_Codeword{3}.Mrep = 1;

% DCI для NPDCCH
grid.Config.Bits.NPDCCH_DCI{4}.bits = ones(1,23);
grid.Config.Bits.NPDCCH_DCI{4}.type = 0;
grid.Config.Bits.NPDCCH_DCI{4}.Mrep = 1;

grid.GridGen();
grid.showResourceGrid;
```

> [!IMPORTANT]
> Метод `GridGen()` необходимо вызывать **после** настройки конфига. Без этого вызова ресурсная сетка не будет сгенерирована, и вы получите ошибку.

### OFDM-модуляция и вывод сигнала

После генерации ресурсной сетки можно выполнить OFDM-модуляцию и визуализировать результат во временной и частотной областях:

```matlab
grid.SignalGen();           % OFDM-модуляция ресурсной сетки

f0 = 900e3;                % Несущая частота, Гц
grid.showSignal(f0);        % Сигнал во временной области
grid.showSpectr(f0);        % Амплитудный спектр
```

Метод `SignalGen()` создаёт объект класса `NBIoTOFDM`, который последовательно обрабатывает каждый OFDM-символ ресурсной сетки: формирует комплексную огибающую из значений поднесущих и добавляет циклический префикс (160 отсчётов для первого символа в слоте, 144 — для остальных). Частота дискретизации составляет 2048 × 15 кГц = 30,72 МГц.

Методы `showSignal(f0)` и `showSpectr(f0)` выполняют перенос сигнала на заданную несущую частоту `f0` и строят соответственно временную диаграмму и амплитудный спектр с отметками границ полосы ±90 кГц.

## Архитектура

Программная реализация построена на нисходящей иерархии классов. Класс `NBIoTResourceGrid` является корневым: он хранит конфигурацию всего сигнала (идентификатор соты `NCellID`, количество фреймов, битовые последовательности для каждого канала, карту цветов для визуализации) и при вызове `GridGen()` создаёт цепочку объектов `NBIoTFrame`, каждый из которых, в свою очередь, содержит десять объектов `NBIoTSubframe`. Каждый сабфрейм определяет свой тип (NPSS, NSSS, NPBCH, NPDSCH, NPDCCH или пустой) через `getSubframeType.m` и генерирует соответствующий участок ресурсной сетки.

Ресурсная сетка представляет собой трёхмерный массив размерности (*k*, *l*, *z*). Первые два измерения — это поднесущие (12 штук, шаг 15 кГц, полоса 180 кГц) и OFDM-символы (14 на сабфрейм, 7 на слот). Третье измерение разделяет сетку на два слоя: первый (*k, l, 1*) содержит комплексные значения сигнала, второй (*k, l, 2*) — цветовые индексы для визуализации.

Перед размещением на ресурсной сетке данные каждого канала проходят через конвейер обработки: CRC → свёрточное кодирование (tail-biting) → согласование скоростей → скремблирование → QPSK-модуляция → маппинг на ресурсные элементы. Конкретные параметры каждого этапа (формула `c_init` для скремблера, количество выходных бит rate matcher и т.д.) зависят от типа канала и определяются стандартом. Распределением данных NPDSCH и NPDCCH по сабфреймам занимаются отдельные планировщики — `NBIoTNPDSCHScheduler` и `NBIoTNPDCCHScheduler` соответственно.

После заполнения ресурсной сетки класс `NBIoTOFDM` выполняет OFDM-модуляцию: преобразует столбцы сетки (по одному на каждый OFDM-символ) в отсчёты временной области с добавлением циклического префикса.

## Структура репозитория

Корневой каталог содержит файл `Main.m` — точку входа, а также класс `NBIoTResourceGrid.m`, который управляет всей генерацией. Директория `@NBIoTFrame` содержит класс фрейма, `@NBIoTSubframe` — класс сабфрейма вместе с функциями генерации каждого канала и сигнала (`gen_NPSS.m`, `gen_NSSS.m`, `gen_NPBCH.m`, `gen_NPDSCH.m`, `gen_NPDCCH.m`, `gen_NRS.m`, а также `getSubframeType.m` и `gridGen.m`).

Отдельные этапы обработки данных вынесены в собственные классы: `@NBIoTScrambler` (скремблирование на базе последовательности Голда длины 31), `@NBIoTQPSK` (QPSK-модуляция), `@NBIoTCRC` (вычисление CRC-16), `@NBIoTEncoding` (свёрточное кодирование с хвостовым сращиванием), `@NBIoTRateMatcher` (перемежение и согласование скоростей). Планировщики `@NBIoTNPDSCHScheduler` и `@NBIoTNPDCCHScheduler` управляют очерёдностью передачи кодовых слов и DCI по своим каналам. Класс `@NBIoTOFDM` реализует OFDM-модуляцию сформированной ресурсной сетки.

Каталог `tests/` содержит юнит-тесты, написанные с использованием MATLAB Unit Testing Framework: `testQPSK.m` проверяет корректность QPSK-модуляции, `testAll.m` — сквозное сравнение результатов текущей реализации с эталонными функциями MATLAB и с предыдущей версией модулей, написанной студентами, работавшими над проектом до нас.

В директории `nb_iot_old/` лежат реализации отдельных модулей (скремблер, модулятор, CRC, кодер Витерби, перемежитель) от предшествующих студентов. Они сохранены для справки и используются для сравнения в `testAll.m`.

Директория `Packs/` содержит git-подмодули `EssentialsPack` и `GF2Pack`, добавленные в ходе экспериментов с подмодулями. В текущей версии проекта они не используются.

## Конфигурация

Все параметры задаются через структуру `grid.Config` до вызова `GridGen()`. Основные параметры: `totalFrames` — количество генерируемых фреймов, `startFrame` — номер стартового фрейма, `NCellID` — идентификатор соты. Битовые последовательности для NPBCH задаются через `Config.Bits.NPBCH`, кодовые слова для NPDSCH — через `Config.Bits.NPDSCH_Codeword{n}` (с полями `.bits` и `.Mrep` для числа повторений), сообщения DCI для NPDCCH — через `Config.Bits.NPDCCH_DCI{n}` (с полями `.bits`, `.type` и `.Mrep`). Маски сабфреймов `Config.NPDSCH.Map` и `Config.NPDCCH.Map` определяют, в каких из десяти сабфреймов фрейма разрешена передача соответствующего канала.

> [!NOTE]
> Сабфреймы 0 и 5 всегда заняты каналами NPBCH и NPSS соответственно, а сабфрейм 9 — сигналом NSSS в чётных фреймах. Значения в `Map` для этих позиций игнорируются.
